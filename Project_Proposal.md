# 智能文件下载管理器 (Smart File Sync) — 项目方案文档

## 一、项目背景

在日常工作和学习中，我们经常需要从网络下载各种文件。操作系统自带的下载工具功能较为单一，缺乏批量管理、并发控制和下载历史追踪等功能。本项目的目标是开发一款基于 Windows 平台的智能文件下载管理器，通过友好的图形界面和丰富的功能，提升文件下载的效率和管理体验。

本项目是《Windows程序设计》课程的综合性实践作业，旨在将课程中学习的 C# 编程、WinForm 窗体设计、并发控制、网络编程、数据库操作等技术综合运用到一个完整的应用程序中。

## 二、功能需求分析

### 2.1 核心功能

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|-------|
| URL 下载 | 支持输入 HTTP/HTTPS 链接下载文件 | 高 |
| 批量下载 | 支持多 URL 批量添加（每行一个） | 高 |
| 并发控制 | 最多 3 个任务同时下载，避免资源耗尽 | 高 |
| 进度显示 | 实时显示下载进度百分比和速度 | 高 |
| 暂停/恢复 | 支持下载任务的暂停和断点续传 | 中 |
| 取消下载 | 支持取消正在进行的下载任务 | 中 |
| 历史记录 | 使用 SQLite 持久化所有下载记录 | 中 |
| 文件管理 | 打开下载目录和已下载文件 | 低 |

### 2.2 非功能性需求

- **性能**：支持同时下载 3 个文件，界面不卡顿
- **可用性**：简洁直观的 Windows 风格界面，中文显示
- **可靠性**：异常处理完善，下载失败有明确提示
- **可维护性**：代码分层清晰（Model-Service-UI），注释充分

## 三、技术选型理由

| 技术 | 选型理由 |
|------|---------|
| **C# 12 + .NET 9** | 课程主要教学语言，最新的 .NET 版本提供更好的异步支持和性能 |

| **xUnit** | 主流的 .NET 测试框架，用于实践 TDD 开发模式 |

## 四、系统架构设计

### 4.1 三层架构

```
┌──────────────────────────────────────────┐
│             表示层 (UI)                    │
│   MainForm / AddDownloadForm / HistoryForm │
├──────────────────────────────────────────┤
│             业务逻辑层 (Services)           │
│   DownloadService / DatabaseService       │
├──────────────────────────────────────────┤
│             数据模型层 (Models)             │
│   DownloadTask / DownloadRecord           │
└──────────────────────────────────────────┘
```

### 4.2 模块关系图

```
用户交互
    │
    ▼
MainForm (主界面) ──→ AddDownloadForm (添加下载)
    │                        │
    │                        ▼
    │                 DownloadService (下载队列管理)
    │                  │              │
    │                  ▼              ▼
    │           DatabaseService   HttpClient
    │                  │              │
    │                  ▼              ▼
    └────────→  SQLite 数据库    远程文件服务器
```

### 4.3 并发设计

- 使用 `ConcurrentDictionary` 管理任务集合，保证线程安全
- 使用 `SemaphoreSlim(3, 3)` 限制最大并发下载数为 3
- 使用 `CancellationTokenSource` 实现任务的暂停和取消
- 使用 `async/await` 异步模式避免 UI 线程阻塞

## 七、RDD 与 TDD 实践说明

### 7.1 RDD（需求驱动开发）

本项目严格遵循需求驱动开发模式：
1. **需求分析** → 先明确功能需求（下载管理、进度显示、历史记录等）
2. **架构设计** → 基于需求设计三层架构（Model-Service-UI）
3. **实现编码** → 按模块逐一实现，每个模块满足对应的需求

### 7.2 TDD（测试驱动开发）

测试先行：

- 先编写 `DownloadTask` 模型的单元测试（属性、状态、格式化方法）
- 再实现模型代码，确保通过测试
- 先编写 `DatabaseService` 的 CRUD 测试（Insert/Update/Delete/Query）
- 再实现数据库服务层
- 测试覆盖率达到 25 个用例，涵盖模型验证、数据持久化和服务逻辑
